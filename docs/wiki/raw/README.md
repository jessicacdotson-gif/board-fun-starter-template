# wiki/raw — kept source material & verified findings

Each file is dated source material we chose to keep: SDK-doc excerpts,
research notes, and findings verified against the real platform.
Rules:

- Date everything; say HOW a claim was verified (docs page, tested on
  hardware, tested in Node, …).
- Docs and reality disagree sometimes — when they do, record both and
  mark `[!CONFLICT]` until resolved (then note the resolution).
- Never edit a finding to say something different — append the
  correction with its own date. Wrong-then-corrected is valuable
  history.

The three `board-*.md` files ship with the template: real findings
from building a real game on Board.fun hardware, scrubbed of that
game's specifics.
